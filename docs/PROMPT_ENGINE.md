# AI Prompt Studio

## Prompt Engine Specification

Version: 1.0

—

# Vision

Every generated prompt should help AI understand the user’s intention clearly.

The prompt should be readable by both humans and AI models.

—

# Core Principle

A good prompt is not merely an instruction.

It is structured communication.

—

# Prompt Structure

Every generated prompt follows this order:

1. Role

2. Goal

3. Context

4. Requirements

5. Constraints

6. Output Format

7. Language

—

# Role

Who should the AI become?

Example:

You are an experienced UX Designer.

—

# Goal

What should the AI accomplish?

Example:

Create a landing page.

—

# Context

Additional background information.

The more relevant context provided, the better.

—

# Requirements

Specific things that must be included.

Example:

- Modern style
- Responsive
- Accessible

—

# Constraints

Things that must NOT happen.

Example:

- Do not use jargon.
- Do not exceed 500 words.

—

# Output Format

Examples:

- Markdown
- HTML
- JSON
- Plain Text

—

# Language

The final output language.

Examples:

- English
- Bahasa Indonesia

—

# Prompt Quality Principles

Every prompt should be:

- Clear
- Specific
- Complete
- Contextual
- Actionable
- Easy to understand

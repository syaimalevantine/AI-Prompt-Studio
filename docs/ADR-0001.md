# ADR-0001

## Title

UKA as the Knowledge Engine of AI Prompt Studio

—

## Status

ACCEPTED

LOCKED

—

## Date

July 2026

—

# Context

AI Prompt Studio began as a template-based prompt builder.

As the product evolved, templates alone became insufficient for supporting semantic intent discovery, prompt consistency, and future scalability.

A structured knowledge layer was required.

—

# Decision

AI Prompt Studio adopts **UKA Official Master Baseline v1.0** as its official Knowledge Engine.

UKA becomes the single source of truth for:

- Intent Library
- Master Domains
- Canonical Concepts
- Context
- Alias
- Knowledge Relationships

Categories are no longer exposed as part of the creator experience.

Categories remain an internal implementation detail.

Creators interact with the Intent Library.

—

# Consequences

## Positive

- Semantic search becomes possible.
- Intent discovery scales without changing the UI.
- Templates become optional.
- Prompt generation becomes knowledge-driven.
- New knowledge is added through UKA instead of hardcoded categories.
- AI Prompt Studio and UKA evolve independently while remaining connected.
- AI Prompt Studio never reads the UKA workbook directly.
- AI Prompt Studio consumes published Knowledge Runtime Packages generated by the Publisher.
- Knowledge Runtime Packages become the runtime contract between UKA and AI Prompt Studio.

-

## Trade-offs

- UKA becomes a required dependency.
- Knowledge quality directly affects prompt quality.
- Canonical governance becomes part of product maintenance.

—

# Implementation Principles

1. UKA is the official Intent Library.

2. Categories are internal only.

3. Search uses:

- Canonical
- Alias
- Context

4. Generator builds prompts from knowledge, not categories.

5. Future knowledge expansion is performed in UKA.

6. AI Prompt Studio consumes published Knowledge Runtime Packages instead of the UKA workbook.

# Internal Publisher Pipeline

The Publisher is internally implemented as a staged build pipeline.

This internal pipeline is an implementation detail and does not
change the architectural contract defined by this ADR.

The Publisher remains the only subsystem authorized to generate
Knowledge Runtime Packages.

The current implementation pipeline is:

```
UKA Official Master Baseline
        │
        ▼
Importer Configuration
        │
        ▼
Workbook Profile
        │
        ▼
Workbook Resolver
        │
        ▼
Workbook Validator
        │
        ▼
Workbook Loader
        │
        ▼
Import Context
        │
        ├───────────────┐
        ▼               ▼
Metadata Builder   Statistics Builder
        │               │
        └───────┬───────┘
                ▼
        Knowledge Builder
                │
                ▼
      knowledge-internal.json
                │
                ▼
Publisher
                │
                ▼
Knowledge Runtime Package
```

## Pipeline Principles

The internal Publisher pipeline follows these principles.

- Every stage has a single responsibility.
- Every stage produces immutable output.
- Builder stages never modify previous stages.
- Validation always occurs before transformation.
- UKA Official Master Baseline remains the single source of truth.
- Knowledge Runtime Packages are generated exclusively by the Publisher.
- AI Prompt Studio consumes only Runtime Packages and never accesses UKA directly.

This pipeline documents the internal implementation only.

The architectural contract defined by this ADR remains unchanged.

# ADR-0001

## Title

UKA as the Knowledge Engine of AI Prompt Studio

—

## Status

ACCEPTED

LOCKED

—

## Date

July 2026

—

# Context

AI Prompt Studio began as a template-based prompt builder.

As the product evolved, templates alone became insufficient for supporting semantic intent discovery, prompt consistency, and future scalability.

A structured knowledge layer was required.

—

# Decision

AI Prompt Studio adopts **UKA Official Master Baseline v1.0** as its official Knowledge Engine.

UKA becomes the single source of truth for:

- Intent Library
- Master Domains
- Canonical Concepts
- Context
- Alias
- Knowledge Relationships

Categories are no longer exposed as part of the creator experience.

Categories remain an internal implementation detail.

Creators interact with the Intent Library.

—

# Architecture

```
Creator
        │
        ▼
Idea
        │
        ▼
Intent Search
        │
        ▼
UKA Knowledge Engine
        │
 ├── Master Domain
 ├── Canonical
 ├── Context
 ├── Alias
 └── Relationships
        │
        ▼
Prompt Generator
        │
        ▼
Generated Prompt
```

—

# Consequences

## Positive

- Semantic search becomes possible.
- Intent discovery scales without changing the UI.
- Templates become optional.
- Prompt generation becomes knowledge-driven.
- New knowledge is added through UKA instead of hardcoded categories.
- AI Prompt Studio and UKA evolve independently while remaining connected.

—

## Trade-offs

- UKA becomes a required dependency.
- Knowledge quality directly affects prompt quality.
- Canonical governance becomes part of product maintenance.

—

# Implementation Principles

1. UKA is the official Intent Library.

2. Categories are internal only.

3. Search uses:

- Canonical
- Alias
- Context

4. Generator builds prompts from knowledge, not categories.

5. Future knowledge expansion is performed in UKA.

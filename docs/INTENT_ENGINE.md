# Purpose

This document defines the Intent Engine architecture of AI Prompt Studio.

The Intent Engine transforms creator ideas into structured runtime knowledge before prompt generation.

The Intent Engine never accesses the UKA workbook directly.

—

# Philosophy

Creators do not think in categories.

Creators think in intentions.

AI Prompt Studio understands those intentions and resolves them into structured knowledge.

Intent is the bridge between creator ideas and the Knowledge Runtime Package.

—

# Intent Definition

An Intent represents what a creator wants to accomplish.

Examples:

- Research
- Presentation
- Business Plan
- Lesson Plan
- Marketing Campaign
- Product Strategy

Intent is not a category.

Categories remain internal implementation metadata.

—

# Knowledge Source

AI Prompt Studio consumes a published **Knowledge Runtime Package**.

The runtime package is generated by the Publisher from:

**UKA Official Master Baseline**

AI Prompt Studio never reads the UKA workbook directly.

—

# Knowledge Runtime Package

The runtime package is composed of independent registries.

- Intent Registry
- Domain Registry
- Canonical Registry
- Relationship Registry
- Runtime Metadata

Every registry is optimized for runtime performance.

—

# Intent Search

Intent Search performs semantic matching.

Search considers:

- Intent Registry
- Canonical Registry
- Alias
- Context

Categories remain internal metadata and are never exposed to creators.

—

# Generator Integration

The Prompt Generator receives:

- Creator Idea
- Intent ID
- Canonical ID
- Context
- Tone
- AI Model
- Output Language

Prompt generation is knowledge-driven and registry-based.

—

# Runtime Principles

The Intent Engine follows these principles.

- Runtime is ID-first.
- Runtime packages are immutable.
- Runtime data is generated, never edited manually.
- UKA remains the single source of truth.
- AI Prompt Studio consumes published runtime packages only.

—

# Future Knowledge Expansion

Knowledge expansion occurs inside UKA.

When UKA evolves:

```
UKA

↓

Publisher

↓

Knowledge Runtime Package

↓

AI Prompt Studio
```

No architectural changes are required inside AI Prompt Studio.

# Levantine Minimal Design System (LMDS)

Version: 1.0.0

—

# Overview

Levantine Minimal Design System (LMDS) defines the visual language and interaction principles of AI Prompt Studio.

The goal is simple:

Create interfaces that feel calm, elegant, lightweight, and effortless to use.

Design should support creativity, never compete with it.

—

# Design Values

## Calm

Interfaces should reduce stress.

No visual clutter.

No unnecessary motion.

—

## Clarity

Every element has a clear purpose.

Users should immediately understand what they can do.

—

## Consistency

Spacing, colors, typography, and interactions should remain predictable throughout the application.

Consistency builds confidence.

—

## Performance

Fast interfaces feel intelligent.

Animations must never reduce responsiveness.

Every transition should be subtle.

—

# Color Palette

## Primary

Purple

Used for:

- Primary buttons
- Active states
- Links
- Focus indicators

—

## Background

Warm Light

Used for:

- Application background

—

## Surface

White

Used for:

- Cards
- Panels
- Dialogs

—

## Text

Dark Gray

Primary reading color.

—

## Secondary Text

Medium Gray

Used for descriptions and helper text.

—

## Success

Green

Used for successful actions and confirmations.

—

## Danger

Red

Used carefully.

Only for destructive actions.

—

# Typography

Primary Font:

System UI Font Stack

Characteristics:

- Highly readable
- Native performance
- Familiar appearance

—

# Border Radius

Small

8px

Medium

16px

Large

24px

Rounded corners should feel soft without appearing playful.

—

# Shadows

Use soft shadows only.

Avoid dramatic elevation.

Interfaces should feel layered, not floating.

—

# Spacing

Base spacing unit:

8px

Recommended scale:

- 8px
- 16px
- 24px
- 32px
- 48px
- 64px

—

# Buttons

Primary Button

- Filled
- Purple
- White text

Secondary Button

- White
- Gray border

Danger Button

- Red

Buttons should always provide visual feedback on hover, focus, and click.

—

# Cards

Cards should include:

- White surface
- Soft shadow
- Medium radius
- Comfortable padding

Cards should never feel crowded.

—

# Icons

Icons should remain simple.

Avoid decorative icons that do not communicate meaning.

—

# Motion

Animations should feel invisible.

Recommended duration:

150–250ms

Use easing that feels natural.

Never animate large areas unnecessarily.

—

# Responsive Design

Desktop

Two-column workspace.

Tablet

Adaptive spacing.

Mobile

Single-column layout.

Touch targets should remain comfortable.

—

# Accessibility

Maintain sufficient color contrast.

Support keyboard navigation.

Keep interactions predictable.

Accessibility is not optional.

—

# Design Philosophy

Good design disappears.

Users should remember what they created,

not the interface they used.
